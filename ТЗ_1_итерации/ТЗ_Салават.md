### **4. ЦЕЛЕВОЕ СОСТОЯНИЕ (MVP)**

#### **4.1. Парсер на Roslyn**
Парсер должен быть переписан с использованием `Microsoft.CodeAnalysis`. Это позволит:
- Разбирать любой корректный C# код
- Получать синтаксическое дерево
- Точно определять приоритеты операторов
- Поддерживать скобки

#### **4.2. Полная арифметика (MVP)**

| Оператор | Приоритет | Пример | Нода |
|----------|-----------|--------|------|
| `*`, `/`, `%` | 1 (высокий) | `x * y` | MultiplyNode, DivideNode, ModuloNode |
| `+`, `-` | 2 (низкий) | `x + y` | AddNode, SubtractNode |
| `( )` | всегда | `(x + y) * z` | — |

**Важно:** В C# операторы `*`, `/`, `%` имеют одинаковый приоритет и выполняются слева направо.

#### **4.3. Сравнения и логика (MVP)**

| Оператор | Тип | Нода |
|----------|-----|------|
| `==` | CompareEqual | EqualNode (уже есть) |
| `!=` | CompareNotEqual | **NotEqualNode (НОВАЯ)** |
| `>` | CompareGreater | GreaterNode (уже есть) |
| `<` | CompareLess | LessNode (уже есть) |
| `>=` | CompareGreaterOrEqual | **GreaterOrEqualNode (НОВАЯ)** |
| `<=` | CompareLessOrEqual | **LessOrEqualNode (НОВАЯ)** |
| `&&` | LogicalAnd | **AndNode (НОВАЯ)** |
| `\|\|` | LogicalOr | **OrNode (НОВАЯ)** |
| `!` | LogicalNot | **NotNode (НОВАЯ)** |

#### **4.4. Условные конструкции (MVP)**

```csharp
if (условие)
{
    // тело
}
else if (условие)
{
    // тело
}
else
{
    // тело
}
```

**Модель в графе:**
- Использовать **цепочку IfNode по ветке False**, как в Roslyn
- IfNode имеет порты: `condition` (вход), `true` (выход на тело if), `false` (выход на следующий if/else)
- ElseIfNode — отдельного узла нет, это просто IfNode, который является продолжением по ветке false
- ElseNode — узел-терминатор, который не имеет условия

#### **4.5. Связи (Edges)**

При парсинге необходимо создавать связи между узлами:

| Пример | Связи |
|--------|-------|
| `z = x + y` | `x` → `AddNode` (порт `inputA`), `y` → `AddNode` (порт `inputB`), `AddNode` → `z` (порт `output`) |
| `if (x > y)` | `x` → `GreaterNode` (`left`), `y` → `GreaterNode` (`right`), `GreaterNode` → `IfNode` (`condition`) |

**Использовать единую схему имен портов (см. приложение).**

#### **4.6. Сохранение имен переменных**

**Правило:**
- Для узлов, которые имеют имя в исходном коде (литералы с объявлением, результат присваивания) → `VariableName = имя`
- Для промежуточных узлов (например, `MultiplyNode` внутри `x + y * z`) → `VariableName = ""` (пустая строка)

| Пример | Узел | VariableName |
|--------|------|--------------|
| `int x = 10;` | LiteralInt | `"x"` |
| `z = x + y;` | AddNode (результат) | `"z"` |
| `x + y * z` | MultiplyNode (промежуточный) | `""` |

#### **4.7. Двунаправленная работа (симметричность)**

Парсер должен быть спроектирован так, чтобы:

1. **Создавать данные, которые генератор сможет прочитать:**
   - Заполнять `NodeData.VariableName` для именуемых узлов
   - Создавать `graph.Edges` с правильными именами портов
   - Для промежуточных узлов оставлять `VariableName` пустым

2. **Читать данные, созданные генератором (через JSON):**
   - При загрузке графа, созданного генератором, корректно восстанавливать структуру

3. **Обеспечивать симметричность:**
   - Код → Ноды (парсер) → Код (генератор) → должен давать код, эквивалентный исходному (с точностью до форматирования)
   - Ноды → Код (генератор) → Ноды (парсер) → должен давать граф, эквивалентный исходному

### **7. ПРИМЕРЫ ДЛЯ ТЕСТИРОВАНИЯ**

#### **Пример 1: Сложная арифметика**
```csharp
int x = 10;
int y = 20;
int z = x + y * 2 % 3;
```

**Ожидаемый результат в данных:**
- LiteralInt: VariableName="x", Value="10"
- LiteralInt: VariableName="y", Value="20"
- LiteralInt: VariableName="", Value="2"
- LiteralInt: VariableName="", Value="3"
- MultiplyNode: VariableName=""
- ModuloNode: VariableName=""
- AddNode: VariableName="z"
- Связи с правильными портами

#### **Пример 2: Условие с логикой (полный фрагмент)**
```csharp
int x = 10;
int y = 20;
int z = 0;
bool flag = true;
if (x >= y && z != 0 || !flag)
{
    z = x + y;
}
else
{
    z = x - y;
}
```

**Ожидаемый результат:**
- IfNode1
  - Condition: OrNode
    - AndNode
      - GreaterOrEqualNode (x >= y)
      - NotEqualNode (z != 0)
    - NotNode (!flag)
  - True: AddNode (z = x + y)
  - False: SubtractNode (z = x - y)
