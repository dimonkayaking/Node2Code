 ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА
🌟 ИДЕЯ ПРОЕКТА
Создать двунаправленный мост между C# кодом и визуальными нодами в Unity.

text
[C# код] ←→ [GraphData] ←→ [Визуальные ноды]
Чтобы разработчики могли:

Писать код → получать визуальный граф

Рисовать ноды → получать готовый C# код

Переключаться между подходами в любой момент

🤔 ПОЧЕМУ ЭТО НУЖНО?
Проблема, которую мы решаем:
В геймдеве есть два лагеря:

Программисты любят код (быстро, точно, мощно)

Дизайнеры/художники любят визуал (наглядно, интуитивно)

Наш плагин — мост между ними:

text
Программист пишет код → дизайнер видит ноды → правит визуально → код обновляется

🔄 КАК ЭТО РАБОТАЕТ
Режим 1: Код → Ноды
text
1. Пользователь пишет:     Debug.Log(1 + 2);
2. Парсер (Роль №1):       превращает в GraphData
3. Интегратор (Роль №3):   конвертирует в VisualData
4. GraphView (Роль №2):    рисует ноды
Режим 2: Ноды → Код
text
1. Пользователь соединяет: [1] → [+] → [Debug]
2. GraphView (Роль №2):    отдает изменения
3. Интегратор (Роль №3):   конвертирует в GraphData
4. Генератор (Роль №1):    создает код: Debug.Log(1 + 2);
