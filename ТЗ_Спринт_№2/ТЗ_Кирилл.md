## 3. Текущие задачи на спринт (4–16 апреля)

### 3.1. Backend-разработка

| № | Задача | Приоритет | Описание |
|---|--------|-----------|----------|
| 1 | `package.json` | 🔴 Критично | Настройка зависимостей проекта (express, jsonwebtoken, bcrypt, pg, cors, dotenv) |
| 2 | `server.js` | 🔴 Критично | Точка входа, запуск сервера, подключение middleware |
| 3 | `.env` | 🔴 Критично | Переменные окружения (PORT, DB_URL, JWT_SECRET) |
| 4 | `config/db.js` | 🔴 Критично | Подключение к PostgreSQL |
| 5 | `controllers/authController.js` | 🔴 Критично | Логика регистрации, входа, генерации JWT |
| 6 | `controllers/progressController.js` | 🔴 Критично | Логика сохранения и получения прогресса пользователя |
| 7 | `routes/authRoutes.js` | 🔴 Критично | Endpoints: `/api/auth/register`, `/api/auth/login` |
| 8 | `routes/progressRoutes.js` | 🔴 Критично | Endpoints: `/api/progress`, `/api/progress/:userId` |
| 9 | `middleware/auth.js` | 🔴 Критично | Проверка JWT-токена |
| 10 | `utils/helpers.js` | 🟡 Важно | Вспомогательные функции (хэширование, валидация) |
| 11 | `tests/auth.test.js` | 🟢 Опционально | Тесты авторизации |

## 4. Критерии готовности

| № | Критерий |
|---|----------|
| 1 | Сервер запускается и обрабатывает запросы |
| 2 | Регистрация и вход работают, JWT-токены выдаются |
| 3 | Прогресс пользователя сохраняется в БД |
| 4 | API защищены middleware (только авторизованные запросы) |
| 5 | Повторное тестирование плагина пройдено, баги закрыты |
| 6 | API протестированы (Postman или автоматические тесты) |
