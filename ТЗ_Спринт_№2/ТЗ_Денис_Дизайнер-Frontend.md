# Техническое задание для Frontend / Дизайнера / Частично Backend (Денис)
## Проект: Сайт-платформа для плагина визуального скриптинга

**Роль:** Frontend-разработчик, Дизайнер, Backend (частично)  
**Срок:** 4–16 апреля (до предзащиты)  
**Статус:** активная фаза разработки

---

## 1. Общая цель

Разработать и подготовить к предзащите полноценный сайт-платформу, включая:
- Полностью рабочий React-фронтенд
- Адаптивный дизайн (десктоп, планшет, телефон)
- Светлую и тёмную темы
- **Часть бекенд-логики** (модели БД, SQL-миграции, API для получения уроков)

## 3. Текущие задачи на спринт (4–16 апреля)

### 3.1. Дизайн и вёрстка

| № | Задача | Приоритет | Описание |
|---|--------|-----------|----------|
| 1 | Фикс масштаба и разметки | 🟡 Важно | Проверить отображение на всех разрешениях |
| 2 | Кнопка «Скачать для проекта Unity» | 🔴 Критично | Заменить две кнопки Windows/Mac на одну универсальную |
| 3 | Пустой экран на входе | 🟡 Важно | Добавить состояние страницы при отсутствии данных |
| 4 | Кнопка «Скачать документацию» | 🔴 Критично | Сделать ссылку на реальный PDF-файл |
| 5 | Валидация полей ввода | 🟡 Важно | Проверка email и длины пароля в формах входа/регистрации |

### 3.2. Backend-часть (модели и API)

| № | Задача | Приоритет | Описание |
|---|--------|-----------|----------|
| 6 | Модели БД | 🔴 Критично | Создать модели `User`, `Lesson`, `Progress` |
| 7 | SQL-миграции | 🔴 Критично | Написать скрипты для создания таблиц |
| 8 | Контроллер уроков | 🟡 Важно | Написать логику получения уроков (GET `/api/lessons`) |
| 9 | Маршруты уроков | 🟡 Важно | Создать endpoints `/api/lessons`, `/api/lessons/:id` |

### 3.3. Интеграция API на фронтенде

| № | Задача | Приоритет | Описание |
|---|--------|-----------|----------|
| 10 | API-сервисы | 🔴 Критично | Создать `api.js`, `authService.js`, `lessonsService.js`, `progressService.js` |
| 11 | Подключение авторизации | 🔴 Критично | Заменить моковые данные на реальные вызовы в `Login.tsx`, `Register.tsx` |
| 12 | Подключение уроков | 🔴 Критично | Загружать уроки из API в `Course.tsx`, `Topic.tsx`, `Lesson.tsx` |
| 13 | Подключение прогресса | 🔴 Критично | Сохранять и загружать прогресс через API в `Profile.tsx` и `Lesson.tsx` |
| 14 | Обновление контекста | 🔴 Критично | Заменить моковые данные в `AppContext.tsx` на реальные API-вызовы |

## 4. Критерии готовности

| № | Критерий |
|---|----------|
| 1 | Сайт корректно отображается на десктопе, планшете и телефоне |
| 2 | Кнопка «Скачать для проекта Unity» скачивает `.unitypackage` |
| 3 | Пустые состояния отображаются без ошибок |
| 4 | Кнопка «Скачать документацию» ведёт к PDF-файлу |
| 5 | Поля ввода проверяют email и длину пароля |
| 6 | Модели БД созданы и описаны |
| 7 | SQL-миграции готовы |
| 8 | API для получения уроков работает |
| 9 | Регистрация и вход работают через API |
| 10 | Прогресс пользователя сохраняется и загружается через API |
