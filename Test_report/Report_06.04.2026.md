# Отчёт о тестировании (дефекты)

### ID‑1: Дублирование консоли
- **Описание**  
  В интерфейсе плагина присутствует консоль, полностью дублирующая стандартную консоль Unity. Назначение этой панели неясно.
- **Серьёзность**  
  Тривиальная (UX)
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  Открыть окно плагина → увидеть панель с логами, идентичную консоли Unity.
- **Рекомендации**  
  Удалить дублирующую консоль или явно объяснить её уникальную пользу (фильтрация логов плагина, цветовая маркировка и т.д.).

### ID‑2: Частичное сохранение графа
- **Описание**  
  Сохраняются использованные ноды, но **не сохраняются связи** между ними.
- **Серьёзность**  
  Критическая
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  1. Создать граф с несколькими нодами и связями.  
  2. Сохранить граф.  
  3. Загрузить граф → связи отсутствуют.
- **Рекомендации**  
  Исправить сериализацию: добавить сохранение всех рёбер (SerializableEdge) с идентификаторами портов и нод.

### ID‑3: Невозможность ручной настройки связей после генерации
- **Описание**  
  При генерации графа из кода связи создаются автоматически, но вручную изменить/настроить их нельзя (инструменты редактирования отсутствуют или заблокированы).
- **Серьёзность**  
  Значительная
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  1. Сгенерировать граф из любого корректного кода.  
  2. Попытаться изменить соединения (перетащить коннектор) → действие не выполняется.
- **Рекомендации**  
  Реализовать полноценное редактирование связей после генерации. Сделать порты редактируемыми, если граф не заблокирован намеренно.

### ID‑4: Ошибка парсинга «ports can't be found»
- **Описание**  
  При парсинге кода возникает ошибка:  
  `The edge can't be properly connected because it's ports can't be found`  
  (с полным стеком вызовов GraphProcessor). Ошибка не блокирует генерацию, но потенциально опасна.
- **Серьёзность**  
  Значительная
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  1. Выполнить парсинг любого кода (например, `int x = 10;`).  
  2. Наблюдать ошибку в консоли.
- **Рекомендации**  
  **Причина для исправления:** ошибка указывает на нарушение целостности графа — соединение ссылается на несуществующие порты. При последующих операциях (сохранении, обратной генерации, клонировании) это может привести к потере данных, падению редактора или неверному коду. Также ошибка засоряет консоль и скрывает действительно важные сообщения.

### ID‑5: Некорректная отрисовка связей
- **Описание**  
  Линия связи сначала уходит назад от выходного порта, затем резко поворачивает к целевому порту (визуальный «изгиб назад»).
- **Серьёзность**  
  Низкая (косметическая)
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  1. Создать граф с двумя нодами.  
  2. Соединить порты.  
  3. Посмотреть на линию связи.
- **Рекомендации**  
  Проверить алгоритм построения кривых Безье (или ломаных) в классе `BaseGraphView`. Корректно задать направление начального и конечного тангенсов.

### ID‑6: Обратная генерация не работает для операции сложения (Add)
- **Описание**  
  Исходный код:  
  ```csharp
  int x = 10;
  int y = 20;
  int z = x + y;
  ```  
  Граф строится правильно, но обратная генерация выдаёт:  
  ```csharp
  int x = 10;
  int y = 20;
  int z = 0;
  ```
- **Серьёзность**  
  Критическая
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  1. Сгенерировать граф из кода с операцией `+`.  
  2. Выполнить обратную генерацию кода из этого графа.  
  3. Сравнить полученный код с исходным.
- **Рекомендации**  
  Исправить логику генерации кода для ноды `Add`: при обратной трансляции значение суммы должно вычисляться, а не подставляться 0.

### ID‑7: Неправильная обработка присваивания переменной переменной
- **Описание**  
  Для кода  
  ```csharp
  int x = 10;
  int y = x;
  ```  
  создаётся дерево, содержащее только ноду `y = 10` (связь с `x` теряется). При последующем переназначении `x` (например, `x = 20;`) создаются пустые численные ноды (без значений).
- **Серьёзность**  
  Критическая
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  1. Вставить указанный код в плагин.  
  2. Проанализировать полученный граф (нода `y` не связана с `x`, а просто получает значение 10).  
  3. Добавить в код строку `x = 20;` и снова сгенерировать граф.
- **Рекомендации**  
  Исправить парсер присваиваний: различать «присвоение значения константы» и «присвоение значения другой переменной». Для `y = x` должна создаваться нода-ссылка на `x`, а не литерал `10`.

### ID‑8: Отсутствие Work flow портов у нод присваивания
- **Описание**  
  Ноды назначения переменной (присваивания) не имеют Work flow входа и выхода. Это делает невозможным последовательное выполнение кода вида:  
  ```csharp
  int x = 10;
  int y = x;
  x = 20;
  ```
  (нет потока управления между присваиваниями).
- **Серьёзность**  
  Значительная
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  1. Создать три ноды присваивания в графе.  
  2. Попытаться соединить их по Work flow (обычный выход в обычный вход).  
  3. Обнаружить отсутствие портов Work flow.
- **Рекомендации**  
  Добавить для каждой ноды присваивания стандартные порты `In` (Exec) и `Out` (Exec), чтобы можно было строить последовательные цепочки. Либо пересмотреть архитектуру — если используется поток данных (dataflow), объяснить, как он заменяет Work flow.

### ID‑9: Избыточные входы у ноды присвоения
- **Описание**  
  У ноды присвоения присутствуют два дублирующих входа: `int value` и `value`. Это сбивает с толку пользователя.
- **Серьёзность**  
  Низкая (UX)
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  Открыть ноду присваивания в графе → увидеть два одинаковых по смыслу входных порта.
- **Рекомендации**  
  Оставить один порт с понятным именем (например, `Input` или `Value`). Удалить дубликат.

### ID‑10: Удалённая нода продолжает появляться в коде
- **Описание**  
  При удалении ноды из графа она **продолжает появляться в коде** при последующей генерации (из того же графа).
- **Серьёзность**  
  Критическая
- **Статус**  
  Открыт
- **Шаги воспроизведения**  
  1. Создать граф из кода (например, с нодой `x = 5`).  
  2. Удалить эту ноду вручную из визуального редактора.  
  3. Выполнить обратную генерацию кода → удалённая нода снова появляется в коде.
- **Рекомендации**  
  При удалении ноды удалять её и из внутреннего `BaseGraph`, и из всех списков/словарей. Проверить, не кэшируется ли удалённая нода в процессе парсинга.
