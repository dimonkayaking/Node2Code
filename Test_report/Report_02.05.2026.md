Отчёт о тестировании Node2Code v22 (промежуточная сборка)

Сборка тестировалась в обычном рабочем процессе: создание графа из кода, ручное добавление нод, работа с подпространствами, очистка/загрузка холста, редактирование кода. Ниже – основные наблюдения.

1. Интерфейс бокового меню  
Боковая панель со списком доступных нод действительно удобна, особенно для новичков – все типы перед глазами, иерархия понятна.  
Предложение: если для каждой ноды добавить короткое описание (хотя бы одну строку) прямо в боковом меню, пользователь сможет ещё быстрее понимать, за что отвечает конкретная нода, не заглядывая в документацию.

2. Проблема с подпространствами новых нод после загрузки графа  
Если загрузить граф из файла, подпространства у загруженных нод открываются нормально. Однако если после загрузки добавить новую ноду (любую, у которой подразумевается подпространство), её подпространство не открывается – соответствующая кнопка/выпадающее меню просто не реактивны или отсутствуют. Воспроизводится стабильно.

3. Ошибка при использовании бокового меню после очистки или загрузки  
При очистке холста специальной кнопкой или после загрузки графа создание нод через боковое меню ломается: при попытке добавить ноду вылетает ArgumentException: «Object at index 0 is null». При этом контекстное меню (правая кнопка мыши по холсту) продолжает работать, ноды создаются без ошибок. Похоже, что очистка/загрузка не сбрасывает какое-то внутреннее состояние, связанное с боковой панелью.

4. Нода «Else» – отсутствует меню подпространства  
У ноды «Else» нет выпадающего меню для открытия подпространства. Из-за этого нельзя детализировать логику ветки «иначе» непосредственно через подграф. Для «If» подпространство работает, а «Else» выпадает из общего паттерна.

5. Некорректное название секции подпространства у ноды «For»  
При открытии подпространства ноды «For» в панели сверху отображается заголовок «Условие» (как будто это граница условия), хотя логически там должна быть «Граница» (либо «Тело цикла» / «For Border»). Это путает, особенно на фоне того, что подпространство действительно содержит условие, но название секции вводит в заблуждение.

6. Проблемы с табуляцией в редакторе кода  
В окне редактирования кода иногда возникает кривое поведение табуляции: символ табуляции вставляется не в позиции курсора, а смещается. Это происходит не всегда, но при активном редактировании заметно мешает – приходится постоянно поправлять отступы вручную.

Открытый вопрос: входы/выходы потока выполнения (execution flow)  
Остаётся нерешённым вопрос с визуальным представлением и логикой входов/выходов execution flow. В текущей версии неясно, как именно будут связываться управляющие переходы между нодами, и каким образом граф будет интерпретировать последовательность выполнения. Это критично для дальнейшей конвертации графа обратно в код, прошу дать пояснения или обозначить план по этому функционалу.

Общее впечатление: плагин уже ощущается цельным, базовый UX продуман, но перечисленные баги мешают полноценно тестировать сценарии с динамическим построением графа. Жду следующую сборку с исправлениями.
