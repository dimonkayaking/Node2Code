# Программа курса: «От визуальной логики к коду: Основы Unity»

## Модуль 1: Первые шаги в Unity и настройка инструментов
**Цель:** Установить Unity, изучить интерфейс редактора и подключить нодовый плагин.

- **Урок 1.1: Установка Unity.**  
  Загрузка Unity Hub, выбор версии редактора (LTS или Unity 6), создание 2D/3D проекта.

- **Урок 1.2: Обзор интерфейса Unity.**  
  Назначение окон: Scene (редактирование сцены), Hierarchy (список объектов), Inspector (параметры выделенного объекта), Project (файлы проекта).

- **Урок 1.3: Подключение плагина визуального программирования.**  
  Установка плагина через Package Manager или Asset Store. Открытие окна графа: холст, панель поиска узлов (нод), инспектор нод.

- **Практика:**  
  *\[Свободное]* Создать на сцене куб, сферу и источник направленного света. Назначить каждому объекту материал с уникальным цветом.

## Модуль 2: Как мыслит Unity. Базовые концепции движка
**Цель:** Изучить жизненный цикл компонента, хранение данных и поток выполнения в визуальном программировании.

- **Урок 2.1: Компоненты и методы жизненного цикла.**  
  Скрипт как компонент. Методы, вызываемые Unity: `Awake` (настройка ссылок при создании), `Start` (перед первым кадром), `Update` (каждый кадр), `FixedUpdate` (фиксированный интервал для физики).

- **Урок 2.2: Переменные (свойства).**  
  Создание переменной типа `float` через интерфейс плагина. Назначение значения по умолчанию.

- **Урок 2.3: Поток выполнения и поток данных.**  
  Разделение понятий: поток выполнения (execution flow) определяет порядок активации узлов, поток данных (data flow) передаёт значения. Связывание переменной скорости с узлом движения внутри `Update`.

- **Практика:**  
  *\[Свободное]* Создать скрипт с двумя переменными: `ИмяОбъекта` (строка) и `Число` (целое). В методе `Start` вывести в консоль сообщение с использованием обеих переменных.

## Модуль 3: Общение объектов
**Цель:** Реализовать взаимодействие между объектами через ссылки и события.

- **Урок 3.1: Получение ссылки на другой объект.**  
  Способы: назначение ссылки через Inspector, поиск по тегу.

- **Урок 3.2: Передача данных между объектами.**  
  Логика: по нажатию мыши на объекте-отправителе изменить цвет объекта-получателя. Использование узлов для отправки событий и изменения свойств компонента.

- **Практика:**  
  *\[Свободное]* Создать объекты «Кнопка» (триггер) и «Дверь». При входе игрока в триггер кнопки дверь должна переместиться вверх на 3 единицы по оси Y.

## Модуль 4: Создаём игру — Space Invaders
**Цель:** Разработать законченный проект, используя только визуальное программирование.

- **Урок 4.1: Игрок и управление.**  
  Настройка корабля. Чтение ввода (клавиатура) через узлы старой системы ввода (`Input.GetAxis`). Перемещение по горизонтали.

- **Урок 4.2: Механика стрельбы.**  
  Создание префаба лазера. Использование узла `Instantiate`. Настройка движения лазера вверх.

- **Урок 4.3: Враги и их логика.**  
  Создание группы врагов. Движение по шаблону: вправо, вниз, влево. Реакция на попадание лазера: уничтожение врага (`Destroy`).

- **Урок 4.4: Подсчёт очков и завершение игры.**  
  Создание менеджера игры (Game Manager). Отправка сигнала от врага к менеджеру при уничтожении. Начисление очков. Проверка условия победы.

- **Практика:**  
  *\[Свободное]* Добавить объект «Укрытие», который разрушается после трёх попаданий вражеских снарядов.

## Модуль 5: Соответствие нод и кода C#
**Цель:** Показать, что визуальные узлы транслируются в код на C#. Ввести понятие потока выполнения в текстовом скрипте.

- **Урок 5.1: Структура простых узлов.**  
  Разбор узлов «Сложение», «Вывод в консоль» и их C#-эквивалентов. Сравнение потока выполнения (последовательность вызовов) и потока данных.

- **Урок 5.2: Генерация кода из графа.**  
  Экспорт C#-скрипта из графа движения игрока (Space Invaders). Разбор: где находятся переменные, метод `Update`, как связи превращаются в последовательные инструкции.

- **Урок 5.3: Синтаксис C#.**  
  Базовые правила: точка с запятой, фигурные скобки, объявление типов (`int`, `float`, `GameObject`), вызов методов.

- **Практика:**  
  *\[Свободное]* Написать простой C#-скрипт, который в методе `Start` выводит в консоль "Hello, Unity!". Затем создать такой же граф из нод в визуальном редакторе.

## Модуль 6: Создание собственных узлов на C#
**Цель:** Расширить возможности плагина, написав кастомный узел на C#.

- **Урок 6.1: Назначение кастомных узлов.**  
  Ограничения готовых узлов. Подготовка шаблона узла в соответствии с API плагина.

- **Урок 6.2: Разработка узла «Удвоитель».**  
  Создание C#-скрипта узла, который принимает число и возвращает его удвоенное значение. Объявление входных и выходных портов.

- **Урок 6.3: Использование узла в проекте.**  
  Добавление кастомного узла в визуальный редактор. Применение в логике начисления очков (например, за красного врага давать удвоенные очки).

- **Практика:**  
  *\[Свободное]* Написать узел «Зигзаг», который принимает скорость и время и возвращает смещение по синусоиде. Использовать его для движения элитного врага.

## Модуль 7: От C# к визуальному графу
**Цель:** Закрепить понимание алгоритмов, перенося логику из текстового скрипта в ноды.

- **Урок 7.1: Создание скрипта в среде разработки (IDE).**  
  Написание простого C#-скрипта: таймер обратного отсчёта с условием `if (time <= 0)`.

- **Урок 7.2: Перенос логики в граф.**  
  Ручное воссоздание логики скрипта с помощью узлов плагина (без автоматической конвертации). Сравнение потоков выполнения в коде и графе.

- **Урок 7.3: Реализация ветвлений и циклов в нодах.**  
  Анализ того, как условие `if/else` представляется в виде узлов ветвления, а цикл `while` — через обратные связи потока выполнения.

- **Практика:**  
  *\[Свободное]* Написать C#-скрипт телепортации объекта при падении ниже координаты Y = -10. Перенести логику в ноды и добавить визуально эффект частиц при телепортации.
