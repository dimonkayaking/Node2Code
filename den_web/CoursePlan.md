# Программа курса: «От визуальной логики к коду: Основы Unity»

## Модуль 1: Первые шаги в Unity и настройка инструментов
**Цель:** Снять страх перед интерфейсом движка и подготовить рабочую среду.

* **Урок 1.1: Установка Unity.** Как скачать Unity Hub, выбрать правильную версию редактора (рекомендуется актуальная LTS или Unity 6) и создать свой первый 2D/3D проект.
* **Урок 1.2: Обзор интерфейса Unity.** Разбор главных окон: Scene (где мы строим мир), Hierarchy (список объектов), Inspector (настройки выбранного объекта) и Project (файлы игры).
* **Урок 1.3: Знакомство с плагином.** Как импортировать ваш плагин в проект. Открытие окна визуального редактора нод, обзор его интерфейса (холст, панель поиска нод, инспектор нод).
* **Практика:**
    * *\[Проверка]* Тест: "Какое окно в Unity используется для изменения свойств конкретного объекта?" (Варианты: Scene, Hierarchy, Inspector, Console).
    * *\[Свободное]* Создать в сцене куб, сферу и источник света, покрасить их в разные цвета с помощью стандартных материалов.

## Модуль 2: Как мыслит Unity. Базовые концепции движка
**Цель:** Понять жизненный цикл скрипта и концепцию хранения данных.

* **Урок 2.1: Что такое скрипт и методы-инициаторы.** Понятие компонента. Разбор главных методов, которые Unity вызывает сама: `Start` (при рождении объекта) и `Update` (каждый кадр). 
* **Урок 2.2: Создание первого свойства (переменной).** Что такое свойство? (коробка для хранения данных). Создаем свойство типа `float` (скорость) через интерфейс плагина.
* **Урок 2.3: Выводим данные в мир.** Связываем созданное свойство скорости с нодой движения внутри метода `Update`. Объяснение того, как данные перетекают по "проводам" от свойства к действию.
* **Практика:**
    * *\[Проверка]* Дано изображение нодового дерева (Свойство `Скорость` = 5 -> Нода `Движение Вперед`). Вопрос: Что произойдет с объектом при запуске игры? (Варианты ответа).
    * *\[Свободное]* Создать скрипт, в котором есть два свойства (Имя объекта и Число). При старте игры (`Start`) объект должен выводить в консоль приветствие, используя эти свойства.

## Модуль 3: Общение объектов
**Цель:** Научить скрипты "видеть" друг друга и взаимодействовать.

* **Урок 3.1: Поиск по тегам и ссылкам.** Как один объект может узнать о существовании другого? Разбор нод получения ссылки (через Inspector или поиск по Tag).
* **Урок 3.2: Передача данных.** Создаем логику: при нажатии кнопки мыши на одном объекте, он меняет цвет другого объекта. Использование нод отправки событий и изменения свойств чужого компонента.
* **Практика:**
    * *\[Проверка]* Тест на логику связей: Как правильно соединить ноду `On Collision Enter` с нодой `Destroy Object`, чтобы уничтожался *тот* объект, с которым мы столкнулись, а не наш?
    * *\[Свободное]* Создать "Кнопку" и "Дверь". Сделать так, чтобы при соприкосновении игрока с кнопкой, дверь перемещалась вверх (открывалась).

## Модуль 4: Создаем игру — Space Invaders
**Цель:** Собрать полученные знания в полноценный проект, используя только визуальное программирование.

* **Урок 4.1: Игрок и управление.** Настройка корабля игрока. Создание нодового дерева для чтения ввода (клавиатура/новый Input System) и перемещения корабля по горизонтали.
* **Урок 4.2: Механика стрельбы.** Как создавать объекты во время игры (нода `Instantiate`). Создание префаба лазера и нодовой логики его полета вверх.
* **Урок 4.3: Враги и их логика.** Создание сетки врагов. Нодовое дерево для движения врагов (вправо, вниз, влево) и реакция на попадание лазера (нода `Destroy`).
* **Урок 4.4: Подсчет очков и финал.** Создание объекта Game Manager. Как враги отправляют сигнал менеджеру при гибели для начисления очков. Условие победы.
* **Практика:**
    * *\[Проверка]* Дано нодовое дерево стрельбы с ошибкой (создается лазер, но не задается его позиция, из-за чего он появляется в центре координат). Вопрос: Почему лазер появляется не из корабля?
    * *\[Свободное]* Добавить в игру "Укрытие", которое разрушается после 3 попаданий вражеских выстрелов.

## Модуль 5: Магия под капотом. От Нод к C#
**Цель:** Демистифицировать код. Показать, что визуальные блоки — это просто красивое представление текста.

* **Урок 5.1: Анатомия ноды.** Построчный разбор простых нод (например, сложение двух чисел или вывод в консоль) и их C# эквивалентов. 
* **Урок 5.2: Читаем код нашей игры.** Генерируем C# скрипт из нашего дерева движения игрока в Space Invaders. Пошаговый разбор: где тут наши свойства (переменные), где метод `Update`, как выглядят наши связи.
* **Урок 5.3: Синтаксис C# для новичков.** Базовые правила языка, которые мы только что увидели: точка с запятой, фигурные скобки, типы данных (`int`, `float`, `GameObject`).
* **Практика:**
    * *\[Проверка]* Сопоставление: Дано 3 картинки визуальных блоков и 3 строчки C# кода. Ученику нужно соединить их попарно.

## Модуль 6: Пишем свою ноду на C#
**Цель:** Научить расширять возможности плагина, используя код.

* **Урок 6.1: Зачем нужны свои ноды?** Ограничения визуального программирования и сила кода. Подготовка шаблона кастомной ноды по стандартам вашего плагина.
* **Урок 6.2: Пишем логику ноды.** Создаем C# скрипт ноды "Сложное зигзагообразное движение" (используя `Mathf.Sin` — математику, которую сложнее собирать блоками). Разбор того, как объявить входы и выходы для ноды.
* **Урок 6.3: Применяем ноду в игре.** Добавляем нашу новую C# ноду в визуальный редактор и назначаем её элитному типу врагов в Space Invaders.
* **Практика:**
    * *\[Свободное]* Написать на C# простую ноду "Удвоитель", которая принимает число, умножает его на 2 и выдает результат. Применить её в логике начисления очков (например, за красного пришельца давать двойные очки).

## Модуль 7: Реверс-инжиниринг: Из Кода в Ноды
**Цель:** Закрепить понимание алгоритмической логики, показав, как компилятор плагина разбирает написанный руками код.

* **Урок 7.1: Пишем скрипт в классической IDE.** Создаем простой C# скрипт с нуля (например, скрипт Таймера обратного отсчета с условием `if (time <= 0)`). 
* **Урок 7.2: Компиляция в дерево.** Конвертируем этот C# скрипт в нодовое дерево с помощью вашего плагина. 
* **Урок 7.3: Анализ графа.** Подробный разбор того, как плагин интерпретировал логику `if/else` (ветвление потока выполнения) и циклы.
* **Практика:**
    * *\[Проверка]* Дано простое C# условие (`if (health < 50) { color = red; }`). Как это будет выглядеть в виде нод? Выбрать правильную схему из предложенных.
    * *\[Свободное]* Написать C# скрипт телепортации объекта при падении ниже определенной координаты Y. Скомпилировать его в ноды и модифицировать визуально, добавив эффект частиц при телепортации.
