## Техническое задание для Салавата и Егора

### Салават – система вкладок подпространств (SubGraph)

**Текущий функционал:** реализованы вкладки для подпространств (Condition, Body, Init, Increment) у нод If, Else, For, While. При открытии вкладки основная вкладка очищается, изменения синхронизируются.

**Необходимо доработать:**

1. **Кнопка-ссылка у каждого подпространства**  
   - Сейчас кнопка "↗" открывает меню выбора подпространства.  
   - Нужно, чтобы у каждой ноды было по одной кнопке на каждое подпространство (например, у If – две кнопки: «Условие» и «Тело»; у For – четыре кнопки: «Объявление», «Граница», «Шаг», «Тело»).  
   - Кнопки должны располагаться в заголовке ноды (рядом с кнопкой сворачивания) или в виде отдельных иконок.

2. **Корректное название вкладок для подпространств цикла for**  
   - Сейчас для Init, Condition, Increment подпространств название вкладки одинаковое («Условие»).  
   - Нужно сделать разные названия: «Объявление», «Граница», «Шаг», «Тело».  
   - Для других нод оставить прежние названия («Условие», «Тело»).

3. **Независимость вкладок от сгенерированного кода**  
   - Сейчас вкладка подпространства открывается только если был произведён парсинг кода (т.е. существует `GraphData`).  
   - Нужно, чтобы вкладка работала всегда, даже в пустом графе. При создании новой ноды её подграфы должны быть доступны для редактирования во вкладке сразу, без генерации кода.

4. **Отображение внутренних нод подпространства в основной вкладке**  
   - Когда пользователь открывает вкладку подпространства, внутренние ноды этой же ноды в основной вкладке пропадают (остаётся только заголовок ноды).  
   - Нужно, чтобы при закрытии вкладки подпространства (или возврате на основную вкладку) внутренние ноды снова отображались в свернутом/развёрнутом виде, с сохранением их позиций.

5. **Авто-синхронизация изменений**  
   - Уже частично реализовано через `SyncSubspaceRuntime`.  
   - Проверить, что при любом изменении во вкладке (добавление/удаление нод, изменение связей, перемещение) содержимое соответствующего `GraphData` подграфа обновляется мгновенно, и при переключении на основную вкладку изменения отображаются корректно.

### Егор – оконтовка нод, сохранение файлов, подсветка кода

**Текущий функционал:** цветная оконтовка нод (по типам), работа кнопок «Сохранить» и «Сохранить как», базовый редактор кода.

**Необходимо доработать:**

1. **Оконтовка нод с подпространствами при генерации из кода**  
   - Когда граф создаётся из C# кода (парсинг), у нод с подграфами (If, For, While, другие тоже проверить) в свёрнутом состоянии пропадает цветная рамка.  
   - Нужно, чтобы цвет оконтовки сохранялся независимо от того, свёрнута нода или развёрнута, и после генерации из кода.

2. **Цвет нод в подпространстве и «старшей» ноды**  
   - При создании ноды внутри подпространства она иногда «забирает» цвет у родительской ноды и родительская теряет цвет.  
   - Необходимо, чтобы каждый узел (нода) имел свой независимый цвет, определяемый его типом, без влияния на другие ноды.

3. **Правильное поведение кнопки «Сохранить»**  
   - **Сценарий А:** пользователь создал новый граф (не загружал файл). При нажатии «Сохранить» должно открываться диалоговое окно «Сохранить как» (как сейчас).  
   - **Сценарий Б:** пользователь загрузил существующий файл (через кнопку «Загрузить» или открыл файл при старте). При нажатии «Сохранить» должно происходить **перезапись того же файла** без диалога.  
   - Такое же поведение должно быть в диалоге при закрытии окна (если есть несохранённые изменения): если файл уже имеет путь – сохранять без запроса имени, иначе – запрашивать путь.

4. **Подсветка синтаксиса C# в редакторе кода**  
   - Нужно **подсвечивать имена переменных/функций, которым соответствуют существующие ноды** (т.е. если в коде встречается имя переменной, совпадающее с `VariableName` какой-либо ноды, то этот текст должен окрашиваться в цвет оконтовки этой ноды).  
   - Подсветка должна обновляться при изменении кода (генерации кода или ручном редактировании).
